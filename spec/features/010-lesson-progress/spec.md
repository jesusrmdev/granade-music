# 010 · Progreso de lecciones

## Objetivo

Permitir que los alumnos marquen las lecciones como completadas y visualicen su progreso por curso. Sin esta funcionalidad el alumno no tiene feedback de cuánto ha avanzado.

## Requisitos funcionales

- Desde la página de una clase (`/clases/[id]`), el alumno puede marcar/desmarcar la lección como completada.
- El estado se persiste en la tabla `lesson_progress` en Supabase.
- En el temario del curso (`/cursos/[slug]`), cada lección muestra un check si está completada y se ve el progreso total (ej. "3/10 lecciones completadas").
- En el dashboard (`/dashboard`), cada curso muestra su progreso (ej. barra o fracción "3/10").
- RLS: el alumno solo puede leer/escribir su propio progreso. Los admins pueden leer todo.

## Criterios de aceptación

- [ ] Botón "Marcar como completada" / "Desmarcar" en la página de clase.
- [ ] El progreso se guarda en `lesson_progress` con RLS.
- [ ] En el temario del curso, las lecciones completadas muestran un check verde.
- [ ] En el temario del curso se muestra "X de Y lecciones completadas".
- [ ] En el dashboard se muestra el progreso de cada curso.
- [ ] Los datos son correctos tras recargar la página.

## Modelo de datos

```sql
create table lesson_progress (
  id           bigint generated by default as identity primary key,
  user_id      uuid not null references auth.users(id) on delete cascade,
  lesson_id    bigint not null references public.lessons(id) on delete cascade,
  completed_at timestamptz not null default now(),
  unique(user_id, lesson_id)
);
```
